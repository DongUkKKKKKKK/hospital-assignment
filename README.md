# 프로젝트 개요
위치 기반 병원 검색 서비스 구현 (3,000개 데이터 최적화 처리)

## 🚀 실행 및 네이버 지도 API 키 설정 가이드

본 프로젝트는 네이버 지도 API를 사용하여 지도를 렌더링합니다. 지도를 정상적으로 확인하기 위해서는 발급받은 **Client ID**를 환경 변수로 설정해야 합니다.

1. 프로젝트 소스코드를 내려받은 후 패키지를 설치합니다: `npm install`
2. 프로젝트 최상단(root) 디렉토리에 `.env.local` 파일을 생성(또는 열기)합니다.
3. 해당 파일에 아래와 같이 본인의 네이버 지도 API Client ID를 입력하고 저장합니다. (따옴표 없이 입력)
   ```env
   VITE_NAVER_MAPS_CLIENT_ID=여기에_본인의_클라이언트_ID를_입력하세요
   ```
4. 터미널에서 `npm run dev` 명령어를 입력하여 개발 환경 서버를 실행합니다.
5. 제공되는 로컬호스트 주소(일반적으로 `http://localhost:5173/`)로 접속하여 화면 우측에 지도가 뜨는지 확인합니다.
   - *팁: 키가 없어도 화면 레이아웃과 병원 목록은 MSW를 통해 정상적으로 확인이 가능하지만, 지도는 렌더링되지 않거나 오류 메시지가 뜹니다.*


## 기술 스택
- React 18

- Naver Maps API

## 기술 스택 선정 이유

**Vite 도입 사유 (동기 및 배경):**
> 기존 프론트엔드 생태계의 표준이었던 Create React App(CRA) 대신 Vite를 빌드 툴로 채택했습니다. 무거운 Webpack 기반의 번들링 방식에서 벗어나, Esbuild를 활용한 압도적인 속도의 사전 번들링과 브라우저의 Native ESM을 이용해 개발 환경 성능을 극대화하고자 했습니다. 대용량 공간 데이터를 다루고 지도를 렌더링해야 하는 본 프로젝트 특성상, 쾌적한 개발자 경험(DX)과 빠른 피드백 루프를 구축하는 것이 중요하다고 판단했습니다.

## 상태 관리 설계 근거
Redux Toolkit을 활용하여 전역 상태를 효과적으로 중앙 집중화하여 관리합니다:
- **사용자 위치 상태:** 사용자의 GPS 좌표 및 지도 중심점 상태를 전역으로 유지하여 컴포넌트 간 불필요한 prop drilling을 방지합니다.
- **병원 목록 상태:** 검색된 병원 목록 데이터를 캐싱하고 관리하여 리렌더링 시에도 지도와 리스트 UI 간의 동기화를 원활히 수행합니다.
- **필터 상태:** 진료 과목 등 다양한 필터 옵션을 스토어에서 관리하여, 필터가 변경될 때마다 병원 목록 데이터와 즉각적인 상호작용이 가능하게 설계했습니다.

## 비동기 처리 모사 (MSW)
MSW(Mock Service Worker)를 활용하여 백엔드 의존성 없이 실제 네트워크 환경과 API를 모사합니다:
- 서비스 워커를 통해 네트워크 레벨에서 요청을 가로채고 모의 데이터(`src/mocks/hospital.json`)를 응답합니다.
- 이를 통해 프론트엔드 개발 시 백엔드 API가 완성되지 않은 상태에서도 병원 탐색 및 필터링 등 모든 핵심 기능을 독립적으로 개발 및 테스트할 수 있도록 구축했습니다.
