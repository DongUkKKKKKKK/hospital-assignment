# 과제 목표

사용자의 현재 위치를 기준으로 주변 병원을 지도에 표시하고 거리순으로
정렬하여 보여주는 웹 애플리케이션을 구현하세요.

# 제공 데이터 및 통신 요구사항

## 1. 제공 데이터

백엔드 없이 진행하기 때문에 병원 데이터는 JSON 파일로 제공합니다.

예시

```json
[
  {
    "id": 1,
    "name": "서울내과의원",
    "lat": 37.5665,
    "lng": 126.978,
    "department": "internal",
    "address": "서울 중구"
  },
  {
    "id": 2,
    "name": "강남정형외과",
    "lat": 37.498,
    "lng": 127.0276,
    "department": "orthopedic",
    "address": "서울 강남구"
  }
]
```

약 3000개의 병원 데이터를 제공합니다.

파일명

    hospital.json

## 2. 비동기 통신 구조

실제 백엔드 API와 통신하는 환경을 모사하여 데이터 페칭 로직을 구현하세요. `hospital.json`을 소스로 사용하되, 아래 중 하나의 방식을 권장합니다.

- **MSW (Mock Service Worker)**: 서비스 워커를 통한 네트워크 요청 가로채기 (권장)
- **JSON Server**: 로컬 API 서버 활용
- **Custom Mocking**: `Promise`와 `setTimeout`을 활용하여 네트워크 지연(Latency)이 포함된 비동기 함수 구현

# 구현 기능

## 1. 지도 표시

- 지도 API를 이용해 지도를 렌더링합니다.
- 애플리케이션 실행 시 지도가 표시되어야 합니다.

---

## 2. 사용자 위치 표시

브라우저의 위치 API를 사용합니다.

    navigator.geolocation

요구사항

- 사용자 위치 표시
- 위치 권한 거부 시 안내 메시지 표시

---

## 3. 병원 마커 표시

hospital.json 데이터를 기반으로

- 병원 위치에 마커 표시
- 마커 클릭 시 병원 정보 표시

표시 정보

- 병원명
- 진료과
- 주소
- 사용자와의 거리

---

## 4. 거리 계산

사용자 위치와 병원 위치 사이의 거리를 계산합니다.

Haversine Formula 등을 사용하여 구현하세요.

예

    distance = calculateDistance(userLocation, hospitalLocation)

---

## 5. 거리순 병원 리스트

지도 옆에 병원 리스트 UI를 구현합니다.

요구사항

- 거리순 정렬
- 병원 클릭 시 지도 이동
- 마커 클릭 시 리스트에서 해당 병원 강조 표시

---

## 6. 진료과 필터

필터 UI를 구현합니다.

예

    전체
    내과
    정형외과
    피부과
    치과

필터 적용 시

- 지도 마커 필터링
- 병원 리스트 필터링

---

## 7. 동일 좌표 병원 처리

여러 병원이 동일 좌표에 존재할 경우 마커가 겹치지 않도록 처리하세요.

가능한 방법

- marker offset
- spiderfy
- marker clustering
